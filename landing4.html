Application of Threat Modelling and Management Techniques

Unit 4

Tools

There are certain features that are critical to be included in threat modelling tools. These features include threat intelligence, the ability to generate reports, ease of system information input, threat dashboards, integration with existing workflow, scalability, a mitigation dashboard, and a rule engine (Mohanakrishnan, 2021). The most common threat modelling tools in 2021 were cairis, IriusRisk, Kenna.VM, Microsoft Threat modelling tool (MTMT), OWASP threat Dragon, SDElements, SecuriCAD, Threagile, Threatmodeler, and Tutamantic. Each tool has its own strengths and weaknesses (Mohanakrishanan, 2021). Cairis - is an open-source tool that allows users to create attacker personas to help identify possible paths to attack but users report it can be time-consuming. OWASP creates flow diagrams based on potential threat lists but because of this often has separate threat dashboards which can decrease usability. One publicly available tool for threat modelling is LINDDUN. LINDDUN stands for link ability, identifiability, nonrepudiation, detectability, disclosure of information, unawareness, and non-compliance. There are six steps which can be found in figure 8 (Shevchenko, 2018). The first step involves building a data flow diagram and then mapping privacy threats. The identified privacy threats are then assigned to specialists' systems and documented as misuse cases which the system uses as threat scenarios (LINDDUN, 2012). LINDDUN and STRIDE are very similar. Archive data can also be used to predict unpredictable changes in threats.

An example of a threat may be using fake web logins to create passwords and usernames. Attackers create a login page on the web which looks identical to the original page it wants to attack and thus the attacker gets users' usernames and passwords. This is an example of a spoofing threat for STRIDE (Nick, 2022). A Denial of services threat example is making sure the configuration page is always accessible even when in high demand. If not, business will suffer. Attackers can change configuration pages online and this can result in crashes (Nick, 2022).
