Practical Applications and Issues in DR Implementations (SRM)

Unit 10

DevSecOps, Human Error and Machine Learning (ML)

DevSecOps is a software development that gives organizations the ability to reduce their development cycles. DevSecOps integrates multiple security practices, so this reduces vulnerabilities to cyber-attacks, reduces issues with different teams, and strengthens test tools and hosting platforms. There are improvements to be made including guidelines and frameworks so DevSecOps can be better understood and thus implemented in the correct way (NIST, 2021).

The human is still the weakest link in information security. If security, especially cyber security was taught as a science in schools early on this may limit the risks of human error in security risk management (Holbrook, 2012)

Machine learning (ML) within security risk management is an important trend to consider. This considers AI and ML. AI and ML are useful tools as cyber criminals are consistently upgrading their tools. AI and ML can do thorough scans on systems but can be limited use with unstructured data (Davies, No Date).
