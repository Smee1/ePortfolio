Introduction to Threat Modelling and Management

Unit 3

Threats and Vulnerabilities

There are differences between threats and vulnerabilities when discussing cyber security. For example, a threat is a potential for a hacker to access your personal information whereas a vulnerability is when there is no protection for that data on the device (Jas, No Date). Understanding harm is also important when considering threat and vulnerability, as direct harm is a threat and chance of harm is a vulnerability e.g., a data breach where medical information is taken is direct harm and vulnerability is the chance of it happening. This is important to consider when I enter the workforce in future companies as I need to be able to identify if it is a threat or vulnerability and how to deal with this. I have read about a few threat modelling techniques. One is the process for Attack simulation and threat analysis (PASTA). PASTA is used by businesses to identify, evaluate and mitigate threats (Cynance, 2022). It can be split into seven stages. Please see figure 6 below (Ranjith, 2020).

There are other types of threat modelling including DREAD and LINDDUN and it all depends on the needs of the organization. PASTA is a logical process but businesses do not like to use it as can have technical risks and takes time to implement (Positive thinking company, 2022). I would likely implement STRIDE as more widely used; however, it is not supported by Microsoft now so this could cause some difficulties in the workplace. There are guides that help with threat modelling. Including OpenFAIR, OCTAVE, TRIKE, VAST ISO, and NIST. Threat modelling is a vital step in risk assessment. Please see figure 7 explained the process (Welekwe, 2022).

OWASP threat modelling playbook is a common cookbook used on GitHub (GitHub, 2021). OpenFAIR is a risk assessment tool commonly used as well. I will use both in the future to assess and analyse threats. Cookbooks are great ways to continually identify and assess threats.
